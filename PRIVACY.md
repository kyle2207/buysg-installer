# buysg 帳密與個人資料處理聲明

> 本文件說明 buysg（股東會紀念品自動下單工具）如何處理你的券商帳密與個人資料。
> 公開版發布於 buysg-installer repo 的 PRIVACY.md（客戶版路徑為 %LOCALAPPDATA%\buysg\home\）；
> 本檔為權威版，改動後需同步過去（同 install.ps1 的同步規則）。

## 一、收集哪些資料、為什麼

buysg 本身**不收集任何資料**。以下資料是「券商官方 API 登入的必要參數」，由你自願填寫、存放在你自己的電腦上：

| 券商 | 資料 | 必要性來源 |
|------|------|-----------|
| 富邦 | 身分證字號、登入密碼、憑證檔（.pfx）、憑證密碼 | 富邦官方 `fubon_neo` SDK 的 `login()` 必要參數（SDK 原生層強制，缺一無法登入） |
| 玉山 | 玉山官方流程產生的 API 設定檔＋憑證 | 玉山官方 `esun_trade` SDK 的登入格式 |

這些是**券商的規定**，任何使用官方 API 的程式（包括你自己寫的）都需要相同資料。buysg 沒有額外收集任何欄位。

## 二、資料如何使用

- **唯一用途**：在你的電腦本機執行時，傳給對應券商的**官方 SDK** 做登入。
- **不會上傳**：帳密不會傳到 buysg 的任何伺服器，不會出現在日誌（logs/）或報表（data/）中。
- **不會代管**：buysg 沒有伺服器保存你的帳密；作者也無法取得。

## 三、資料如何保存

- 帳密只存在你電腦的 `certificates/` 資料夾內。
- `buysg doctor` 驗證通過後，會自動用 **Windows DPAPI**（Chrome 儲存密碼的同一套系統機制）將機密欄位加密改寫：
  - 加密綁定「這台電腦＋這個 Windows 使用者」，檔案被複製到其他電腦或其他使用者帳號下**無法解密**。
  - 寫入前有解密回讀比對（roundtrip 驗證），確保加密不會毀損你填的資料。
- 想更換帳密：把該欄位改回明文存檔，下次執行 doctor 會再自動加密。

## 四、buysg 會連線的所有端點

buysg 的全部網路行為僅限以下四類，你可以用防火牆或封包工具自行驗證：

1. **券商官方 API**（富邦/玉山）—— 登入、查庫存、下單，帳密只會出現在這裡
2. `yustockgifts.com` —— 下載公開紀念品清單（不帶任何個人資料）
3. Supabase（零股悠帳號）—— 登入授權檢查（只有你的零股悠帳號 token，無券商帳密）
4. GitHub Releases —— `buysg update` 下載更新（不帶任何個人資料)

## 五、buysg 絕不做的事

- ❌ 上傳、回傳、備份你的券商帳密或憑證
- ❌ 在你不知情的情況下登入你的帳戶（只有你執行 `buysg` 時才登入你勾選的帳戶）
- ❌ 對非你本人提供帳密的帳戶做任何存取
- ❌ 收集下單紀錄、庫存、損益等交易資料到任何伺服器

## 六、你的責任與選擇

- 帳密由你**自願填寫**；不填寫，該帳戶不會被列入執行清單，其餘功能照常可用。
- 請妥善保管你的電腦與 Windows 使用者密碼——DPAPI 加密在「同機同使用者」下是可解密的，這是讓程式能自動登入的必要前提。
- 本工具僅供操作**你本人（或已獲授權管理）的證券帳戶**；請勿填入未經授權的他人帳密。
